# Pass criteria — Use case implications present

The synthesis must connect findings to the stated use case — what the product team should do or investigate next, specific to what the corpus revealed. Generic recommendations not tied to the corpus fail this eval.

## Criteria (binary)

1. ✅ / ❌ The synthesis contains an "Implications" section (or equivalent) that goes beyond restating themes.
2. ✅ / ❌ At least one implication is specific to this corpus — it would be different if the corpus were different (test: if this exact implication would appear in a generic SaaS synthesis, it fails).
3. ✅ / ❌ Implications connect to the stated use case in the fixture's synthesis task (e.g., activation funnel improvements, AI feature design, investment decision).
4. ✅ / ❌ Implications do not overstate confidence from the corpus — they don't say "users want X" when only 1 of 4 participants expressed that.

## Failure modes this catches

- Implications list: "1. Improve onboarding. 2. Reduce friction. 3. Make it easier to get started." — generic SaaS recommendations that don't reflect this corpus.
- Implications ignore the most specific finding (e.g., Interview 2's "sample data set" insight for the activation funnel fixture) in favor of generic framing.
- Implications overstate: "Users clearly want role-specific paths" based on one participant (Interview 4 in activation funnel).
- Sparse corpus produces confident implications as if the 3 short interviews were enough to direct product investment.

## Notes for the grader

The specificity test for criterion 2: replace the corpus with a different set of interviews for a different product. Would the same implication appear? If yes, it's generic. If no — if it's specific to these participants, this product, this use case — it passes.

For the sparse corpus, the implication should be "more research is needed" with specific priority questions — not a direction. A synthesis that recommends product investment from 3 thin interviews has overconfidence.
