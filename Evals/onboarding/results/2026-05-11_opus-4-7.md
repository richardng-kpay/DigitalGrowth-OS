**Fixture:** riley-park-minimalist

**Quoted transcript evidence (lines 104–124):**

>
> 1. P0 this week?
> 2. P1 soon?
> 3. P2 / backlog?
> 4. Blockers + owners?

>

> **Assistant:** Consolidated read-back:
> - **P0:** run intro 1:1 with Jordan Singh; map product surface to 1-page doc
> - **P1:** set up tracking on landing page conversion; meet with eng lead
> - **P2:** none
> - **Blockers:** no baseline dashboard (owner TBD — eng lead?)
> - **Follow-ups:** none
>
> All correct?
>
> **User:** All correct. Eng lead will own the dashboard question once we meet.

The transcript itself acknowledges the violation at lines 128 and 412–414.

**Introspection hypothesis (why did the assistant produce this?):**
The assistant appears to have over-applied the Minimalist persona's "terse, checklist-first, low ceremony" directive to a Phase 4 step where the interactivity contract is supposed to be persona-independent. Once the persona was locked in at Phase 2 ("Effects: ... `/peer-review` light edit ... terse, checklist-first"), the assistant treated "low ceremony" as license to compress five separate per-category confirmations into a single consolidated checklist. This is a known cross-cutting failure mode: persona affects voice/tone, but should not affect workflow-rule per-step interactivity.

A second contributing factor: the user fixture itself uses sparse data ("P2: nothing. Follow-ups: none yet"), so the marginal value of per-category confirmation feels low and the assistant's local-optimum heuristic kicked in. The workflow rule exists precisely to defend against this — empty categories should still get an individual "is that right?" ask, both to catch missing items and to preserve the contract.

**Recommended harness fix:**
- Add a phase-level guardrail in `Workflows/interactive-onboarding.md` that says "Phase 4 read-back MUST be one ask per category, regardless of persona. Minimalist persona affects message length, not message count." Place this explicitly adjacent to the persona-effects matrix.
- Consider a structural enforcement (e.g., a Phase 4 sub-state machine with required transitions p0_confirm → p1_confirm → p2_confirm → blockers_confirm → followups_confirm) so the model cannot legally collapse the asks.
- Add a fixture-level test: include a Phase 4 fixture where one category contains a typo that only per-category read-back will surface.
