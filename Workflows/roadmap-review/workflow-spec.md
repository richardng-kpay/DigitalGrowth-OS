# Workflow: Roadmap & Deliverable Review

**Use when:** Weekly check-in on roadmap progress and deliverable status
**Trigger phrase:** "roadmap review"
**Cadence:** Weekly (suggest: Monday or Friday)
**Output:** Updated tasks, flagged risks, clear priorities for the week

---

## Steps

### Step 1: Pull Current State

Read:
- `Tasks/active.md` — what's in flight
- `Tasks/backlog.md` — what's queued
- `Projects/[your-main-project]/brief.md` — milestone status
- `GOALS.md` — 30-60-90 day targets

---

### Step 2: Review Milestones

For each active milestone in the project brief:

| Milestone | Target Date | Status | On Track? | Notes |
|-----------|-------------|--------|-----------|-------|
| [Milestone 1] | [DATE] | | | |
| [Milestone 2] | [DATE] | | | |
| [Milestone 3] | [DATE] | | | |
| [Milestone 4] | [DATE] | | | |

**Status key:** 🟢 On Track | 🟡 At Risk | 🔴 Blocked

---

### Step 3: Identify Risks & Blockers

For each item that is 🟡 or 🔴:

- What's the root cause?
- What's needed to unblock?
- Who do I need to talk to?
- Is a timeline adjustment needed?

---

### Step 4: Reprioritize Active Tasks

1. Archive completed items in `Tasks/active.md`
2. Pull in next-up items from backlog
3. Ensure active tasks are aligned to the nearest milestone
4. Confirm: Am I working on the highest-leverage thing right now?

---

### Step 5: Prepare for Manager

If a weekly update or 1:1 is coming up:
- Note 2–3 key updates to share
- Flag any decisions you need from them
- Draft using the relevant section of `Templates/weekly-performance-report.md`

---

## Review Cadence

| Review Type | Frequency | Who |
|-------------|-----------|-----|
| Personal roadmap review | Weekly | Solo |
| Manager 1:1 alignment | Weekly | You + manager |
| Stakeholder update | Monthly | You + team |
| Quarterly OKR review | Quarterly | You + manager |
